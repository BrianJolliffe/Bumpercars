1️⃣ Numeric Columns (Revenue, Score, Quantity, Price)

Most common: First click = Descending

Why?

Users usually want to see largest values first

Example use cases:

Top revenue

Highest score

Most recent date

Largest quantity

Highest priority

So:

First click → High → Low
Second click → Low → High
Third click → Unsorted (optional)

This is very common in:

Analytics dashboards

Admin panels

Financial tools

SaaS reporting interfaces

2️⃣ Dates

Almost always: Descending first (Newest → Oldest)

Users usually want:

Most recent transactions

Latest updates

Recent activity

So descending is expected behavior here.

3️⃣ Text Columns (Name, Company, Category)

Most common: First click = Ascending (A → Z)

Why?

Alphabetical order convention

Easier to scan

Matches spreadsheet behavior

So:

First click → A → Z
Second click → Z → A
4️⃣ Status / Priority Columns

This depends on what makes sense for the product.

Sometimes:

Highest priority first

“Active” before “Inactive”

Custom business logic sort

Here, clarity matters more than convention.
